Fix branch target too far in large procedures (GitHub issue #17). The compiler generates bra (relative branch, +-127 byte range) for all control flow. Large loops exceed this range. The fix: replace all bra/brt/brf with long-branch sequences using la+jmp. For unconditional branches (bra Lxx), emit 'la r0,Lxx; jmp (r0)'. For conditional branches (brt Lxx / brf Lxx), keep the conditional branch but invert it to skip over a long jump: e.g., for 'brt Lxx' emit 'brf _skip; la r0,Lxx; jmp (r0); _skip:'. Search codegen.h for all emit of bra/brt/brf and replace with long-branch helpers. This affects cg_if, cg_do_while, cg_do_count, and possibly cg_binop comparison branches.
