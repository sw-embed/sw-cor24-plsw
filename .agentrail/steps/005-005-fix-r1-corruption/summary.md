Fixed GitHub #31. Root cause: plsw prologue used 'add sp,-frame_sz' which is signed 8-bit (-128..127); for frames >127 bytes the immediate silently wrapped (-204 -> +52), growing sp and causing nested-call pushes to overwrite the saved fp/r2/r1 slots. The outer unwind then popped zeros and jmp'd to PC=0, re-entering _start. Fix: emit 'sub sp,frame_sz' (opcode 0xa2, 24-bit immediate) when frame_sz > 127. sno_exec had a 204-byte frame that triggered it. Verified by removing the inline-ASM bra-self halt workaround from sno_main.plsw and confirming fizzbuzz.sno and factorial.sno both halt cleanly via the normal return path. plsw commit 0bf3ebe; snobol4 cleanup commit 71e3db6; closed issue #31.
