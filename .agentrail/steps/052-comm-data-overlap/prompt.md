Fix .comm directives overlapping with .data string literals (GitHub issue #12). The COR24 assembler uses a single linear address counter and ignores .data/.text markers, so .comm directives placed after .data labels get addresses that overlap with string literal data. The fix (Option A from the issue): convert .comm to explicit .byte 0,0,... entries so they get proper unique addresses in the .data section. Find where .comm is emitted in codegen.h (likely in cg_emit_static_data) and change it to emit .byte 0 entries instead. For 'DCL X(6) BYTE;' (no INIT), instead of '.comm _X, 6', emit '_X:' followed by '.byte 0,0,0,0,0,0'. Add a test verifying the output uses .byte instead of .comm.
