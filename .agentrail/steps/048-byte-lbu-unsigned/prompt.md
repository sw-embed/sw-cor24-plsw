Fix BYTE field reads to use lbu instead of lb (GitHub issue #8). The compiler generates lb (sign-extending load) for BYTE fields, but BYTE is unsigned -- values >= 128 get sign-extended to negative 24-bit values. Find all places in codegen.h where BYTE field reads emit lb and change them to lbu. This likely affects cg_deref_load (ptr->field reads), cg_field_load (record.field reads), and possibly cg_load_var (simple BYTE variable reads). Search for 'lb ' in codegen.h emission. The fix: when the field width is 1 byte (fw == 1), emit lbu instead of lb. Add a test that verifies lbu is generated for BYTE field access.
