Fixed MACRODEF+DCL coexistence in .msw includes (GitHub #2). Root cause: mac_parse_clause rejected keyword tokens (PTR, INT) as clause names, only accepting TOK_IDENT. Fixed by accepting any token with text as clause name. Also added error propagation from mac_parse_def to parse_program. Added test for mixed DCL+MACRODEF in one include. Verified SNOBOL4 project had to split descr.msw into descr.msw + descr_macros.msw as workaround.
