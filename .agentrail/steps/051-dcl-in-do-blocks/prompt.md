Support DCL inside nested DO blocks or emit a clear error (GitHub issue #11). Currently, DCL inside a DO block silently fails -- the variable is parsed but not laid out, causing 'undefined variable for store' later. The simplest fix (Option 1): treat DCL inside DO blocks the same as proc-level DCL -- allocate stack space and add to the current scope. The parser already parses DCL in statement position (parse_stmt handles it). The issue is likely in layout_locals which only walks top-level statements in the body block, not nested DO block children. Investigate layout_locals in layout.h to understand how it walks the AST. If DCL nodes inside nested DO blocks are skipped, fix the walk to recurse into DO block children. Add a test that compiles a proc with DCL inside a DO block.
