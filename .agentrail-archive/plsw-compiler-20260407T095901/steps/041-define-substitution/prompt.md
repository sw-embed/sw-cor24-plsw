Investigate and implement %DEFINE value substitution in expressions (GitHub issue #1). Currently %DEFINE NAME VALUE only works for %IF DEFINED(NAME) but does not substitute VALUE when NAME appears in expressions. The goal is compile-time constants with zero runtime cost. Investigate the lexer's existing %DEFINE handling, then implement Option A: make %DEFINE NAME VALUE perform text substitution at the lexer level when NAME appears as an identifier token. When the lexer produces an identifier, check the define table; if found and the define has a value, replace the identifier token with the value token. Update the chain example or add a new example demonstrating %DEFINE constants. Reference: PL/I uses %DCL/%ACTIVATE but a simpler single-step approach is preferred for PL/SW.
