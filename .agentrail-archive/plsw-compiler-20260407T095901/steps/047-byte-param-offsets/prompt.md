Fix BYTE procedure parameters using wrong stack offsets (GitHub issue #7). The calling convention pushes all arguments as 3-byte values via `push`, but layout_params calculates parameter offsets using the declared type width (1 byte for BYTE). This causes later parameters to read from wrong stack locations. The fix: in layout_params (layout.h), always use 3-byte (word) slots for parameter offset calculation regardless of declared type. BYTE params should still be read with lb (load byte) but at the correct 3-byte-aligned offset. Find layout_params in layout.h, change the parameter width calculation to always use 3 bytes minimum. Add a test that compiles a procedure with mixed PTR/BYTE/INT parameters and verifies correct offsets in the generated assembly.
